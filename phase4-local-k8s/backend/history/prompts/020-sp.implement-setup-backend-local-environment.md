# Setup Backend Local Environment

## Overview
This document records the process of setting up the local backend environment for the Phase II Hackathon II Todo App.

## Process Performed

### 1. Created Virtual Environment
- Created a Python virtual environment in `backend/venv`
- Command: `python -m venv backend/venv`

### 2. Installed Required Dependencies
- Activated the virtual environment
- Installed the following packages:
  - fastapi
  - uvicorn
  - sqlmodel
  - python-jose[bcrypt]

### 3. Set Environment Variable
- Set DATABASE_URL to use local SQLite: `DATABASE_URL=sqlite:///./test.db`

### 4. Created Startup Scripts
- Created `run_backend.bat` for Windows Command Prompt
- Created `run_backend.ps1` for PowerShell

## Files Created
- `backend/venv/` - Python virtual environment
- `run_backend.bat` - Batch script to run the backend server
- `run_backend.ps1` - PowerShell script to run the backend server

## How to Run the Backend
After the environment is set up, you can run the backend server in two ways:

### Method 1: Using the provided scripts
- On Windows: Run `run_backend.bat` or `run_backend.ps1`

### Method 2: Manual activation
1. Activate the virtual environment:
   - On Windows Command Prompt: `backend\venv\Scripts\activate.bat`
   - On PowerShell: `backend\venv\Scripts\Activate.ps1`
2. Set the environment variable: `set DATABASE_URL=sqlite:///./test.db`
3. Run the server: `uvicorn backend.main:app --reload --app-dir .`

## Result
The backend environment is now ready to run locally with a SQLite database for development purposes.
